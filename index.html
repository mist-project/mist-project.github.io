What is Mist

Mist is an image preprocessing tool for the purpose of protecting the style/content of the images from being mimicked by state-of-the-art AI-for-Art applications, including Stable Diffusion, Novel.ai, and scenario.gg. Mist does this by adding human-invisible watermarks to images thus making them unrecognizable for the model used by AI-for-Art applications. Concretely speaking, if AI-for-Art applications try to mimic these Misted images, the output image of such mimicry will be a mess.

Advantages & Examples

Mist is strong against various AI-for-Art applications, robust to noise purification, and time-efficient. Examples are given to prove that Mist provides state-of-the-art protecting watermarks for images with robustness to noise purification. Furthermore, the watermarking process costs only several minutes!

Download

Mist can be downloaded by two approaches. For users, you can download the file from Huggingface. After unzipping the file, Mist can be boosted by clicking on the executable file Mist. In another case, if you are a developer and want to build Mist from scratch, you may download the source code of Mist from Github and check the Readme file for installation. Currently, Mist supports Linux and Windows.

Validate

Mist works on preventing AI-for-Art applications to mimic the style/content of the image. Users can validate the effectiveness of Mist in two steps. First, add watermarks to images with Mist, Second, feed these images to AI-for-Art applications and synthesize new images based on their mimicry functions, for example, textual inversion and Dreambooth. We provide guidelines for such validation in our documentation.

FAQ close

Q: What is the computing resource requirement to run Mist?
A: We provide a table in our documentation about the device requirement. Also, we strongly recommend users who lack computing resources to use the low VRAM mode of Mist. This would greatly reduce the computing resource requirement.
Q: Would Mist protect the image absolutely?
A: No. We have validated the performance of Mist on Stable Diffusion, Novel.ai, and scenario.gg, which cover the mainstream AI-for-Art applications that support the mimicry function. However, even the most popular operating system is not absolutely safe. For us, new challenges may be originated from newly released AI-for-Art applications, novel mimicry functions supported, or newly proposed methods to purify our watermarks. What we can do is continuously listening to our users who would find bugs in the system and fixing them accordingly. That's why we build a Discord for both developers and users for the response. Join us now!
Q: What should I do if my device cannot run Mist even under low VRAM mode?
A: We feel sorry for the limitation and would like to share that we are going to push Mist as an open-source serverless online service, which will allow all users to use Mist, even if they do not have a computer. Please wait for us for a while! -If I use an attack whose strength is smaller than recommended to make the watermark not so obvious, would Mist still work? -Of course. The recommended strength is to promise an overall performance under different mimicry cases (e. g. Textual inversion and Dreambooth in Stable Diffusion, Img2Img in Novel.ai, etc). While reducing the strength, Mist will still work with relatively moderate performance. Moreover, if the user only cares about some of the mimicry cases, he or she can choose a mode in Mist that will specifically work on those cases and reduce the watermark strength to better demonstrate the image. For the question of how to choose the mode, we give detailed instructions in our documentation.
